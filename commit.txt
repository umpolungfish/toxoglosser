fix: Resolve all build issues and compilation errors in Toxoglosser toolkit

This commit resolves all critical build issues that were preventing successful compilation
of the Toxoglosser toolkit, enabling both basic and enhanced versions to build properly.

## Build Issue Resolutions

### 1. Duplicate Constant Declarations
- **Issue**: Multiple packages had duplicate Windows constants like MEM_COMMIT_RESERVE, PAGE_READWRITE, etc.
- **Fix**: Created centralized `core/constants.go` file with all Windows constants to eliminate duplication across packages
- **Impact**: Eliminates compiler errors due to redeclared constants

### 2. Undefined Function References  
- **Issue**: `undefined: common.ptrToString` errors throughout the codebase
- **Fix**: Capitalized `ptrToString` to `PtrToString` to properly export the function from the common package, and updated all references to use the exported function
- **Impact**: Resolves all undefined function errors in core, evasion, utils, and other packages

### 3. Syscall Parameter Type Mismatches
- **Issue**: `NtProtectVirtualMemory` calls had incorrect parameter types causing compilation failures
- **Fix**: Corrected parameter types to match function signature - `NtProtectVirtualMemory(processHandle windows.Handle, baseAddress *uintptr, regionSize *uintptr, newProtect uint32, oldProtect *uint32)`
- **Impact**: Fixes all memory protection related syscall calls across all files

### 4. Package Import Issues
- **Issue**: Missing `golang.org/x/sys/windows` import in toxoglosser_enhanced.go causing `undefined: windows` errors
- **Fix**: Added required Windows package import to files that reference Windows-specific functions
- **Impact**: Resolves undefined Windows API references

### 5. Function Redefinition Conflicts
- **Issue**: `copyMemory` function defined in both `evasion/patch.go` and `evasion/unhook.go` causing package-level redeclaration
- **Fix**: Renamed function in unhook.go to `copyMemoryInternal` to avoid conflicts
- **Impact**: Eliminates package-level function redeclaration errors

### 6. Constant Reference Updates
- **Issue**: Main files (toxoglosser.go, toxoglosser_enhanced.go) using constants that were moved to centralized location
- **Fix**: Updated all constant references to use the `core` package (e.g., `MEM_COMMIT_RESERVE` â†’ `core.MEM_COMMIT_RESERVE`)
- **Impact**: Ensures all constant references work with the centralized definition approach

### 7. Unused Variables and Imports
- **Issue**: Various unused imports and variables causing build failures
- **Fix**: Removed unused imports and variables throughout the codebase
- **Impact**: Eliminates `imported and not used` and `declared and not used` errors

## Build Process Improvements

### Cross-Compilation Support
- Verified cross-compilation workflow for Linux to Windows builds
- Updated build instructions to reflect proper environment variable usage
- Ensured CGO_ENABLED=1, GOOS=windows, GOARCH=amd64, and CC=x86_64-w64-mingw32-gcc settings work correctly

### Go Version Compatibility
- Updated go.mod to use Go 1.21 (was causing build issues with older Go versions)
- Ensured all dependencies are compatible with Go 1.21
